# LMForge v0.1 — Architecture & Usage Walkthrough

LMForge is a hardware-aware local LLM orchestrator. It probes your hardware once (`lmforge init`), selects the optimal engine (oMLX on Apple Silicon, SGLang on Linux/NVIDIA, llama.cpp on CPU/small GPU), and manages a pool of models — chat, embedding, and re-ranking — all served through a unified OpenAI-compatible API.

---

## Architecture

```
lmforge start
      │
      ▼
 EngineManager (supervision loop)
      │
      ├─── ModelSlot: chat model    (port 11431)  ─► oMLX / SGLang / llama.cpp
      ├─── ModelSlot: embed model   (port 11432)  ─► same engine, ModelRole::Embed
      └─── ModelSlot: rerank model  (port 11433)  ─► llama.cpp only (--reranking)
      │
      ▼
 API Server (port 11430)
      ├── POST /v1/chat/completions   → proxied to chat slot
      ├── POST /v1/embeddings         → proxied to embed slot (batched)
      ├── POST /v1/rerank             → proxied to rerank slot (sigmoid-normalised)
      └── GET  /v1/models             → index with capabilities + dims
```

**Key design constraints:**
- **One engine per platform.** oMLX on macOS, SGLang on Linux/NVIDIA. No cross-engine mixing.
- **Per-role engine startup.** `ModelRole` (Chat/Embed/Rerank) is derived from model capabilities and injected into the engine at spawn time (e.g. `--embeddings`, `--reranking` flags).
- **VRAM-aware LRU eviction.** Models are unloaded in least-recently-used order when VRAM is full.

## Quick Start

```bash
# 1. Initialise (hardware probe + engine install + catalog seeding)
lmforge init

# 2. Pull a chat model
lmforge pull qwen3.5:4b

# 3. Start the daemon
lmforge start

# 4. Chat via OpenAI-compatible API
curl http://localhost:11430/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"qwen3.5:4b","messages":[{"role":"user","content":"Hello!"}]}'
```

## Embedding Models

### Pull and use

```bash
# See what's available
lmforge catalog                          # current platform format (mlx/gguf)
lmforge catalog --search embed           # filter to embed models only
lmforge catalog --format gguf           # force gguf format

# Pull an embedding model
lmforge pull nomic-embed-text:v1.5      # macOS/MLX
lmforge pull qwen3-embed:4b             # macOS/MLX, 4B, strong multilingual
lmforge pull nomic-embed-text:v2        # Linux/GGUF

# The daemon auto-detects the model role — no flags needed
# Just call /v1/embeddings with the model ID
curl http://localhost:11430/v1/embeddings \
  -H 'Content-Type: application/json' \
  -d '{"model":"qwen3-embed:4b","input":"Hello world"}'
```

### Response format (OpenAI-compatible)

```json
{
  "object": "list",
  "data": [{ "object": "embedding", "index": 0, "embedding": [0.012, -0.034, ...] }],
  "model": "Qwen3-Embedding-4B-4bit-DWQ",
  "usage": { "prompt_tokens": 3, "total_tokens": 3 }
}
```

### Batch chunking

Large requests are automatically split into batches of `embed_batch_size` (default 32, configurable):

```toml
# ~/.lmforge/config.toml
[orchestrator]
embed_batch_size = 64   # adjust for your model and VRAM
```

### Dimension auto-detection

On the first successful `/v1/embeddings` call, LMForge reads the vector length from the response and stores it in `~/.lmforge/models.json`. Subsequent `lmforge models list` and `GET /v1/models` calls show the correct `DIMS` column without any manual configuration.

### Qdrant integration example

```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams
import httpx

LMFORGE = "http://localhost:11430"
MODEL = "qwen3-embed:4b"
DIMS = 1024   # truncated via Matryoshka — saves ~75% storage vs full 4096

client = QdrantClient(":memory:")
client.create_collection("docs", vectors_config=VectorParams(size=DIMS, distance=Distance.COSINE))

def embed(texts: list[str]) -> list[list[float]]:
    resp = httpx.post(f"{LMFORGE}/v1/embeddings", json={"model": MODEL, "input": texts})
    resp.raise_for_status()
    data = resp.json()["data"]
    # Truncate to DIMS (Matryoshka)
    return [item["embedding"][:DIMS] for item in sorted(data, key=lambda x: x["index"])]

# Index documents
docs = ["The Eiffel Tower is in Paris.", "The Colosseum is in Rome."]
vectors = embed(docs)
client.upsert("docs", points=[
    {"id": i, "vector": v, "payload": {"text": d}}
    for i, (v, d) in enumerate(zip(vectors, docs))
])

# Query
query_vec = embed(["Where is the Colosseum?"])[0]
results = client.search("docs", query_vector=query_vec, limit=2)
```

## Re-ranking Models (llama.cpp platforms only)

### Pull and use

```bash
lmforge pull qwen3-reranker:0.6b   # fast, 600M
lmforge pull bge-reranker-v2-m3    # strong, 560M

curl http://localhost:11430/v1/rerank \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "qwen3-reranker:0.6b",
    "query": "What is quantum computing?",
    "documents": ["Quantum computers use qubits...", "Classical computers use bits..."],
    "top_n": 2,
    "return_documents": true
  }'
```

### Response format (Cohere/Jina schema — compatible with LangChain, LlamaIndex)

```json
{
  "model": "qwen3-reranker:0.6b",
  "results": [
    { "index": 0, "relevance_score": 0.94, "document": { "text": "Quantum computers use qubits..." } },
    { "index": 1, "relevance_score": 0.31, "document": { "text": "Classical computers use bits..." } }
  ],
  "usage": { "prompt_tokens": 48, "total_tokens": 48 }
}
```

Scores are normalised to `[0, 1]` via `sigmoid()` so they're consistent regardless of whether the engine returns raw logits.

### LangChain integration

```python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import CohereRerank

reranker = CohereRerank(
    base_url="http://localhost:11430",
    cohere_api_key="lmforge",   # any non-empty string
    model="qwen3-reranker:0.6b",
    top_n=5,
)
retriever = ContextualCompressionRetriever(base_compressor=reranker, base_retriever=your_retriever)
```

## Model Management

```bash
# List installed models with capabilities
lmforge models list

# Example output:
# MODEL                                      FORMAT   ENGINE   CHAT   EMBED  RERANK  THINK  DIMS        SIZE
# ──────────────────────────────────────────────────────────────────────────────────────────────────────────────
# qwen3.5:4b                                 mlx      omlx     ✓      –      –       –      –         2.5 GB
# qwen3-embed:4b                             mlx      omlx     –      ✓      –       –      4096      2.2 GB
# qwen3-reranker:0.6b                        gguf     llamacpp –      –      ✓       –      –         380 MB

# Browse available shortcuts
lmforge catalog                       # platform default format
lmforge catalog --format gguf         # GGUF only
lmforge catalog --search rerank       # filter

# Remove a model from disk
lmforge models remove qwen3-embed:4b

# Unload active model from VRAM (keeps files)
lmforge models unload
```

## HuggingFace Token (Gated Models)

Some models require access approval on HuggingFace. After requesting access on the model page:

```bash
export HF_TOKEN=hf_your_token_here
lmforge pull meta-llama/Llama-3.1-8B-Instruct   # or any gated model
```

LMForge reads `HF_TOKEN` (or `HUGGING_FACE_HUB_TOKEN`) automatically. The token is never stored to disk.

## Configuration Reference

`~/.lmforge/config.toml`:

```toml
port = 11430
bind_address = "127.0.0.1"
log_level = "info"
default_chat_model = "qwen3.5:4b"
default_embed_model = "qwen3-embed:4b"

[orchestrator]
keep_alive = "5m"          # unload idle models after 5 minutes
max_loaded_models = 3      # max concurrent models in VRAM (0 = unlimited)
embed_batch_size = 32      # max inputs per /v1/embeddings batch chunk

[resources]
max_gpu_memory_fraction = 0.75
min_free_disk_gb = 10
```

## API Reference Summary

| Endpoint | Method | Description |
|---|---|---|
| `/v1/chat/completions` | POST | OpenAI chat — rejected for embed/rerank models (400) |
| `/v1/completions` | POST | OpenAI legacy completions |
| `/v1/embeddings` | POST | OpenAI embeddings — rejected for chat models (400), batched automatically |
| `/v1/rerank` | POST | Cohere/Jina re-ranking — 501 on oMLX/SGLang platforms |
| `/v1/models` | GET | Lists installed models with capabilities and `embedding_dims` |
| `/lf/model/unload` | POST | Unload a specific model from VRAM |
| `/lf/engine/status` | GET | Engine and slot status |
| `/lf/config` | GET/POST | Read/update config (live, no restart needed) |
